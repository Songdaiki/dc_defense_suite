# DC Defense Suite

디시인사이드 마이너갤 운영용 통합 방어 확장입니다.  
댓글 삭제, 게시글 도배기 분류, 반고닉 도배기 분류, IP 차단, 게시물/댓글 자동 감시를 한 확장에서 운영할 수 있도록 만든 MV3 크롬 확장입니다.

이 저장소 루트 자체가 확장프로그램 폴더입니다.

## 핵심 기능

### 수동 기능

- `댓글 방어`
  - 유동닉 댓글 자동 삭제
  - 페이지 범위, 요청 딜레이, 사이클 딜레이, 동시 처리 수 설정 가능
- `게시글 분류`
  - 유동 게시글을 도배기 탭으로 자동 분류
  - 마이너갤의 도배기탭 번호를 이용해 `headtext` 변경
- `반고닉 분류`
  - 식별코드 작성자의 누적 게시물/댓글 비율을 조회해 도배기 후보를 판정
  - 기준을 넘는 작성자가 쓴 글을 도배기 탭으로 분류
- `IP 차단`
  - 도배기 탭에 있는 유동 게시물을 기준으로 운영 차단
  - 차단 목록을 다시 스캔해서 내가 자동 차단한 대상 수동 해제 지원

### 자동화 기능

- `게시물 자동화`
  - 게시물 급증 + 유동 비율 급증 감지
  - 공격 상태 진입 시 `게시글 분류 + IP 차단` 자동 ON
  - 종료 조건 충족 시 자동 OFF, 해당 세션의 자동 차단 IP 자동 해제
- `댓글 자동화`
  - 1~N페이지 게시물의 `reply_num` 순증가를 감시
  - 임계치 초과 시 `댓글 방어` 자동 ON
  - `새 댓글 수`와 `실제 검증 삭제 수`가 함께 낮아지면 자동 OFF

## 대상 / 전제

- `디시인사이드 마이너갤` 전용입니다.
- 운영 권한이 있는 관리자/부매니저 사용을 전제로 합니다.
- 공통 설정의 `갤러리 ID`, `도배기탭 번호`는 직접 맞춰야 합니다.
- `게시글 분류`, `반고닉 분류`, `IP 차단`, `게시물 자동화`는 도배기탭이 존재하는 갤에서만 의미가 있습니다.

## 설치

1. 이 저장소를 내려받습니다.
2. 크롬에서 `chrome://extensions`를 엽니다.
3. 우측 상단 `개발자 모드`를 켭니다.
4. `압축해제된 확장 프로그램을 로드합니다`를 누릅니다.
5. 이 저장소 루트 폴더를 선택합니다.

대상 폴더:

```text
/home/eorb915/projects/dc_defense_suite
```

Windows에서 로드할 경우, 해당 폴더가 보이는 실제 경로를 선택하면 됩니다.

## 초기 설정

팝업 상단 `공통 설정`에서 아래 두 값을 먼저 저장합니다.

- `갤러리 ID`
  - 예: `thesingularity`
- `도배기탭 번호`
  - 예: `130`

중요:

- 공통 설정은 `모든 기능이 정지 상태`일 때만 변경할 수 있습니다.
- 공통 설정을 바꾸면 관련 기능 상태와 통계가 초기화됩니다.
- 설정값은 `chrome.storage.local`에 저장되며 팝업을 닫거나 브라우저를 재시작해도 유지됩니다.

## 기능별 동작

### 1. 댓글 방어

목록 페이지를 순회하면서 댓글이 있는 글만 대상으로 잡고, 각 글의 댓글을 조회해 유동닉 댓글을 삭제합니다.

주요 설정:

- `시작 페이지`
- `끝 페이지`
- `요청 딜레이`
- `사이클 딜레이`
- `동시 처리 수`

특징:

- 삭제 요청은 실제 관리자 댓글 삭제 API를 사용합니다.
- 삭제 후 댓글을 다시 읽어 `실제 검증 삭제 수`를 계산합니다.
- `댓글 자동화`가 실행 중일 때는 수동 조작이 잠깁니다.

### 2. 게시글 분류

목록 페이지에서 일반 게시물 중 유동 작성자 글만 골라 도배기 탭으로 분류합니다.

특징:

- `공지`, `설문`, 숫자 글번호가 아닌 행은 분류 대상에서 제외합니다.
- 이미 도배기 탭인 글은 다시 분류하지 않습니다.
- `headtext` 변경 API를 사용합니다.

### 3. 반고닉 분류

목록 페이지에서 `data-uid`가 있는 작성자를 모은 뒤, 작성자 레이어 API로 누적 게시물/댓글 수를 조회합니다. 게시물 비율이 높은 식별코드 작성자를 도배기 후보로 보고 현재 페이지의 글을 도배기 탭으로 분류합니다.

기본 판정 기준:

- `최소 총 활동수 >= 20`
- `최소 게시물 비율 >= 90%`

특징:

- 같은 `uid`는 한 사이클에서 한 번만 조회합니다.
- `판정 UID`는 고유 `uid` 기준으로 집계합니다.
- `중지` 시 in-flight 분류 요청은 abort 시도 후 상태를 정리합니다.

### 4. IP 차단

도배기 탭 페이지를 순회하며 유동 게시물을 자동 차단합니다.

특징:

- 차단은 게시물 번호 기준 관리자 차단 API를 사용합니다.
- 수동 해제 버튼은 차단 목록 페이지를 다시 읽어 내부 해제 ID를 추출한 뒤 해제 API를 호출합니다.
- `총 차단`은 고유 활성 차단 기준으로 집계되며, 같은 대상을 중복으로 다시 잡았다고 계속 증가하지 않도록 보정되어 있습니다.

### 5. 게시물 자동화

새 게시물 수와 새 유동 게시물 비율을 기준으로 공격을 감지하고, 자동으로 `게시글 분류`와 `IP 차단`을 제어합니다.

기본 흐름:

1. 첫 폴링은 기준 스냅샷만 저장
2. 이후 `새 게시물 수`, `새 유동 게시물 수`, `유동 비율` 계산
3. 공격 조건을 연속 충족하면 `ATTACKING`
4. `게시글 분류 ON`, `IP 차단 ON`
5. 종료 조건을 연속 충족하면 `RECOVERING`
6. `게시글 분류 OFF`, `IP 차단 OFF`
7. 해당 공격 세션에서 자동 차단한 IP만 자동 해제

기본 설정:

- `감시 폴링 시간(ms)` 기본 30000
- `감시 페이지 수` 기본 1
- `공격 감지 새 게시물 수` 기본 50
- `공격 감지 유동 비율(%)` 기본 85
- `공격 감지 연속 횟수` 기본 2
- `공격 종료 새 게시물 수` 기본 10
- `공격 종료 유동 비율(%)` 기본 40
- `공격 종료 연속 횟수` 기본 3

중요:

- 실행 중에는 `게시글 분류`, `반고닉 분류`, `IP 차단` 수동 조작이 잠깁니다.
- `감시 페이지 수`는 실행 중 변경할 수 없습니다.

### 6. 댓글 자동화

1~N페이지 게시물의 `reply_num` 순증가를 감시해 댓글 도배 징후가 보이면 자동으로 `댓글 방어`를 켭니다.

기본 흐름:

1. 첫 폴링은 기준 스냅샷 저장
2. 이후 `새 댓글 수`, `변화 글 수` 계산
3. 공격 조건 연속 충족 시 `댓글 방어 ON`
4. 종료 조건 연속 충족 시 `댓글 방어 OFF`
5. 1 poll reseed 후 다시 `NORMAL`

기본 설정:

- `감시 폴링 시간(ms)` 기본 30000
- `감시 페이지 수` 기본 2
- `공격 감지 새 댓글 수` 기본 250
- `공격 감지 연속 횟수` 기본 2
- `공격 종료 새 댓글 수` 기본 50
- `공격 종료 실제 삭제 수` 기본 50
- `공격 종료 연속 횟수` 기본 2

중요:

- `새 댓글 수`는 `reply_num` 순증가 기준입니다.
- 실제 총 댓글량이 아니라 현재 감시 범위(기본 1~2페이지)에서 관측된 순증가 합계입니다.
- 실행 중에는 수동 `댓글 방어` 조작과 `댓글 자동화` 통계 초기화가 잠깁니다.

## 권장 운영 순서

### 기본 수동 운영

1. 공통 설정 저장
2. 상황에 맞는 수동 기능 ON
3. 필요 시 로그 확인
4. IP 차단 기능은 공격 종료 후 `내가 차단한 대상 해제`

### 자동화 운영

1. 공통 설정 저장
2. 수동 기능이 모두 정지 상태인지 확인
3. `게시물 자동화` 또는 `댓글 자동화` ON
4. 로그로 `SEEDING -> NORMAL -> ATTACKING -> RECOVERING` 흐름 확인

## 제한사항 / 주의사항

- 마이너갤 전용입니다.
- 요청량이 많은 편입니다. `댓글 방어 + 게시글 분류 + 반고닉 분류 + IP 차단`을 동시에 켜면 DC 쪽 `403/429` 가능성이 올라갑니다.
- 자동화는 내부적으로 수동 엔진을 child scheduler처럼 제어합니다. 자동화가 켜진 동안 관련 수동 기능은 일부 잠깁니다.
- `댓글 자동화`의 `새 댓글 수`는 실제 전체 댓글량이 아니라 감시 범위의 `reply_num` 순증가 합계입니다.
- `반고닉 분류`는 `data-uid`가 있는 작성자를 대상으로 하며, 디시 응답 구조가 바뀌면 판정 실패 로그만 남기고 스킵할 수 있습니다.
- `반고닉 분류`의 게시물/댓글 수는 현재 갤 한정이 아니라 작성자 누적 활동 수로 해석하는 전제를 사용합니다.
- `중지` 버튼을 눌렀을 때도 이미 나간 HTTP 요청은 브라우저/서버 타이밍에 따라 완전히 즉시 취소되지 않을 수 있습니다. 다만 현재 구현은 로컬 상태가 잘못 완료 처리되지 않도록 보강되어 있습니다.

## 문제 해결

### 설정 저장 시 `알 수 없는 feature`가 뜨는 경우

대부분 크롬이 구버전 service worker를 들고 있는 상태입니다.

1. `chrome://extensions`
2. `DC Defense Suite` 새로고침
3. 팝업 닫았다가 다시 열기

### 입력값이 저장 전에 원래 값으로 되돌아가는 경우

현재 버전은 draft 보호가 들어가 있어서, 수정 후 `설정 저장` 전까지는 입력값이 자동으로 덮어써지지 않습니다. 만약 이상하면 확장을 새로고침한 뒤 다시 확인하세요.

### IP 해제가 일부만 되는 경우

차단 목록 페이지를 다시 읽어 내부 해제 ID를 찾는 구조입니다. 차단 목록 페이지 수가 많으면 `해제 검색 페이지 수`를 늘려야 할 수 있습니다.
