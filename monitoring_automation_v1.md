# DC Defense Suite - 감지 자동화 v1 정책

## 목적
- 평상시에는 수동 운영을 유지한다.
- 비정상적인 게시글 도배 공격이 감지되면 자동으로 대응 모드에 진입한다.
- 공격이 종료된 것으로 판단되면 자동 대응을 종료하고, IP 차단도 자동 해제한다.

## 적용 범위
- 대상: 마이너 갤러리
- 감시 대상: 게시글 목록
- 자동 대응 대상:
  - 게시글 분류
  - IP 차단

## 감시 주기
- 30초마다 게시글 목록을 폴링한다.

## 감시 방식
- 감시 대상 페이지 범위를 주기적으로 읽어 현재 게시글 스냅샷을 만든다.
- 직전 스냅샷과 현재 스냅샷을 비교해 `이번 폴링 구간에 새로 등장한 게시물`만 추린다.
- 새 게시물 수는 `현재 스냅샷에는 있고 직전 스냅샷에는 없던 postNo 개수`로 계산한다.
- 유동 비율은 `새 게시물 집합` 안에서 `data-ip`가 있는 게시물 비율로 계산한다.

## 기본 감시 페이지 정책
- 기본값은 `1페이지`로 한다.
- 1페이지에 약 50개 게시물이 보이는 현재 운영 환경에서는 공격 감지용으로 충분하다고 본다.
- 매우 짧은 시간에 1페이지 수용량을 초과하는 도배가 발생하면 실제 새 글 수는 과소 집계될 수 있다.
- 다만 v1의 목적은 정확한 총량 집계보다 `공격 여부 감지`이므로, 임계치를 넘는지 판정하는 용도로는 1페이지 기본값을 허용한다.
- 필요 시 감시 페이지 수는 설정값으로 늘릴 수 있어야 한다.

## 핵심 지표
- 최근 30초 내 새 게시글 수
- 최근 30초 내 새 게시글 중 유동 게시글 비율

## 상태 정의
- `NORMAL`
  - 평상시 상태
  - 자동 대응 비활성화
- `ATTACKING`
  - 공격 감지 상태
  - 자동 대응 활성화

## 공격 감지 조건
아래 조건을 2회 연속 만족하면 `ATTACKING`으로 전환한다.

- 최근 30초 내 새 게시글 수 `50개 이상`
- 최근 30초 내 유동 게시글 비율 `85% 이상`

## 공격 감지 후 동작
`ATTACKING` 진입 시 즉시 아래를 수행한다.

- 게시글 분류 `ON`
- IP 차단 `ON`

## 공격 종료 조건
아래 조건을 3회 연속 만족하면 공격 종료로 판단한다.

- 최근 30초 내 새 게시글 수 `10개 미만`
- 최근 30초 내 유동 게시글 비율 `40% 미만`

## 공격 종료 후 동작
공격 종료로 판단되면 즉시 아래를 수행한다.

- 게시글 분류 `OFF`
- IP 차단 `OFF`
- 자동으로 차단 해제 절차 시작

## IP 차단 해제 정책
- 해제 대상은 자동 대응으로 차단한 대상만 포함한다.
- 해제는 현재 저장된 자동 차단 추적 목록을 기준으로 수행한다.
- 해제 방식은 기존 수동 해제 로직과 동일하게 차단 목록 페이지를 스캔한 뒤 내부 해제 ID를 찾아 호출한다.

## 설정값으로 노출할 항목
- 감시 폴링 시간(ms)
- 감시 페이지 수
- 공격 감지 새 게시글 수 임계치
- 공격 감지 유동 비율 임계치(%)
- 공격 감지 연속 만족 횟수
- 공격 종료 새 게시글 수 임계치
- 공격 종료 유동 비율 임계치(%)
- 공격 종료 연속 만족 횟수

## v1 기본값
- 감시 폴링 시간: `30000ms`
- 감시 페이지 수: `1`
- 공격 감지 새 게시글 수: `50`
- 공격 감지 유동 비율: `85%`
- 공격 감지 연속 만족 횟수: `2`
- 공격 종료 새 게시글 수: `10`
- 공격 종료 유동 비율: `40%`
- 공격 종료 연속 만족 횟수: `3`

## 운영 가정
- 현재까지 관찰된 패턴상 공격은 약 1시간 전후 지속된다.
- 공격 종료 후 1~2분 내 재공격하는 패턴은 아직 확인되지 않았다.
- 따라서 종료 조건을 충분히 엄격하게 두면 자동 해제까지 포함해도 운영 리스크가 낮다고 본다.

## v1 의도
- 복잡한 이상탐지보다 명확한 임계치 기반 상태 전환에 집중한다.
- 오탐보다 실전 대응 속도를 우선한다.
- 향후 v2 이상에서 제목 유사도, IP 반복도, 알림 모드, 수동 승인 모드 등을 추가할 수 있다.
