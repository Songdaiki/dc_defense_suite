# 역류기 오탐 케이스 노트

## 목적

최근 확인된 역류기 오탐 케이스를

1. 어떤 경로로 매칭됐는지
2. dataset 문제인지 matcher 문제인지
3. 이번 패치로 해결된 것과 아직 남아 있는 것을

바로 구분할 수 있게 정리한다.

---

## 현재 matcher 경로

역류기 공용 matcher는 아래 3단계로 동작한다.

1. exact
2. permutation
3. containment

쉽게 말하면:

- `exact`
  - 공백/보이지 않는 filler를 정리한 뒤 완전히 같은 문자열이면 매치
- `permutation`
  - 긴 문구에서 글자 구성을 정렬해 비교
  - 문장부호 차이만 있는 변형을 같은 것으로 잡는다
- `containment`
  - 긴 문구의 공통 조각 3개를 signature로 만들어 비교
  - 순서가 조금 섞여도 긴 재활용 제목을 잡기 위한 경로다

---

## 케이스별 원인

### 1. `어어 그러지마라`

- 원인: `exact`
- dataset 원문:
  - `어어 그러지 마라`
- 왜 매치됐나:
  - 정규화 후 둘 다 `어어그러지마라`가 된다.

예시:

- dataset: `어어 그러지 마라`
- 실제 댓글: `어어 그러지마라`
- compare key: `어어그러지마라`
- 결과: exact hit

정리:

- matcher 버그가 아니라 dataset에 너무 일반적인 문구가 들어간 케이스다.
- 이후 공용 dataset에서 제거했다.

---

### 2. `ㅇㄱㅈㅉㅇㅇ?`

- 원인: `exact`
- dataset 원문:
  - `ᄋᄀ ᄌᄍᄋᄋ?`
  - `ᄋᄀᄌᄍᄋᄋ?`
- 왜 매치됐나:
  - 공백 제거 후 같은 compare key가 된다.

정리:

- 이것도 matcher 버그가 아니라 dataset 오염 케이스다.
- 짧은 초성/자모 문구는 dataset 정리 대상이다.
- 이후 공용 dataset에서 제거했다.

---

### 3. `대량실업이 온다......`

- 원인: `permutation`
- dataset 원문:
  - `대량실업이 온다...`
- 왜 매치됐나:
  - permutation은 문장부호를 빼고 글자 multiset만 비교한다.
  - 둘 다 핵심 글자 구성이 `대량실업이온다`로 같다.

예시:

- dataset: `대량실업이 온다...`
- 실제 글: `대량실업이 온다......`
- permutation core: 둘 다 `대량실업이온다`
- 결과: permutation hit

정리:

- 점 개수 차이만 있는 변형을 잡으려는 의도와 맞는 동작이다.
- 이 케이스를 풀려면 permutation 정책 자체를 바꿔야 한다.
- 다만 현재는 dataset 원문 `대량실업이 온다...`를 공용 dataset에서 제거해서
  runtime 기준으로는 더 이상 이 케이스가 `true`가 되지 않는다.

---

### 4. `속보) 오푸스 4.7 입갤 ㅋㅋㅋㅋㅋㅋㅋㅋㅋㅋㅋㅋㅋ`

- 원인: `containment`
- dataset 원문:
  - `속보) 오푸스 4.1 입갤 ㅋㅋㅋㅋㅋㅋㅋㅋㅋㅋㅋㅋㅋ`

패치 전에는 containment anchor 조합에서 아래처럼 공통 조각이 성립했다.

- `속보)오푸스`
- `입갤ㅋㅋㅋㅋ`
- `ㅋㅋㅋㅋㅋㅋ`

문제는 마지막 조각 같은 순수 자모 반복이 너무 흔하다는 점이다.

즉 실제로는

- `4.1`
- `4.7`

처럼 중요한 숫자가 다르더라도,
앞부분과 웃음 자모 조각이 겹치면 같은 제목처럼 오탐될 수 있었다.

### 이번 패치

containment 조각 후보에서 아래 같은 **순수 한글 자모 반복 조각**을 제외한다.

- `ㅋㅋㅋㅋ`
- `ㄷㄷㄷㄷ`
- `ㅅㅂㅅㅂ`
- `ㅇㅇㅇㅇ`

핵심은:

- `입갤ㅋㅋㅋㅋ`처럼 실제 단어가 붙은 조각은 남긴다.
- `ㅋㅋㅋㅋㅋㅋ`처럼 자모만 있는 조각만 뺀다.

패치 후 기대 결과:

- `속보) 오푸스 4.1 입갤 ...`
- `속보) 오푸스 4.7 입갤 ...`

이 둘은 이제 공통 anchor 3개를 만들기 어려워져서 containment 오탐이 크게 줄어든다.

---

### 5. `눈을 떴구나`

- 현재 공용 title dataset matcher 기준:
  - `false`
- 즉 현재 코드만 놓고 보면 이 문구는 post/comment 공용 matcher에서 바로 삭제되지 않는다.

다만 과거 댓글 전용 source 파일에는 아래 항목이 있었다.

- `눈을 떴구나`
- `눈을 떴구나. 미정갤로 오거라.`
- `눈을 떴구나...`

정리:

- 현재 runtime 공용 matcher 기준 삭제 원인은 아님
- 만약 실제 현장에서 지워졌다면
  - 예전 빌드 상태
  - 또는 search duplicate 경로
  - 또는 다른 필터 경로
  를 같이 봐야 한다.

## 이번 패치 범위

이번 패치는 **containment 조각 후보 축소**만 건드린다.

바뀌는 것:

- `ㅋㅋㅋㅋ` 같은 자모-only 조각은 containment 공통조각으로 사용하지 않음

안 바뀌는 것:

- exact
- permutation
- dataset 원문 자체

containment 패치만으로는 아래 케이스가 남았지만,
현재는 dataset 정리까지 같이 반영해서 runtime 기준으로 제거되었다.

- `어어 그러지마라`
- `ㅇㄱㅈㅉㅇㅇ?`
- `대량실업이 온다......`

정리:

- `어어 그러지마라`
- `ㅇㄱㅈㅉㅇㅇ?`
- `대량실업이 온다......`

이 셋은 matcher보다는 dataset 정리 또는 permutation 정책 조정 이슈였고,
현재 배포본 manifest version `2026-04-16-222009` 기준으로는 dataset에서 제거된 상태다.

---

## 후속 정리 우선순위

1. containment 오탐 억제
   - 이번 패치 완료
2. dataset 짧은 일반문구 청소
   - 예: `어어 그러지 마라`, `ㅇㄱㅈㅉㅇㅇ?`
3. permutation 허용범위 재조정
   - 예: 점 개수만 다른 문구를 계속 같은 것으로 볼지 결정 필요
