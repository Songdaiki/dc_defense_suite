## 1. 이 문서가 의미하는 것

이 문서는 공개 CSV가 아니라 **공식 클라이언트가 실제로 받은 DAT feed**를 복호해서 정리한 결과다.

쉽게 말하면:

- 공개 CSV 문서: 누구나 보는 공개 목록
- 이번 문서: 공식 클라이언트가 `xd.x1.client.api.vpngate2.jp/api/?session_id=...` 에서 실제로 받은 200개 목록

복호 흐름도 아래 순서다.

1. DAT body에서 raw payload를 분리한다.
2. payload 앞 20바이트를 SHA-1 한다.
3. 그 SHA-1 값을 RC4 키로 써서 payload 본문을 푼다.
4. outer PACK에서 `data`, `timestamp`, `sign` 등을 읽는다.
5. inner PACK에서 실제 서버 200개를 읽는다.

## 4. 주요 필드 뜻

- `Name`: 짧은 서버 이름. 예: `public-vpn-51`
- `IP`: 실제 접속 대상 IP. 예: `219.100.37.13`
- `CountryShort / CountryFull`: 국가 코드와 국가명. 예: `JP` / `Japan`
- `HostName / Fqdn`: 공식 feed가 준 호스트명과 FQDN
- `Score`: 공식 feed 내부 점수. 보통 높을수록 상단 우선순위에 가까움
- `PingToJapan / PingToGoogle`: 지연 관련 수치. 낮을수록 보통 유리
- `SslPorts`: SSL-VPN 포트 문자열. 예: `443`
- `OpenVpnUdpPorts`: OpenVPN UDP 포트 문자열. 비어 있으면 미광고
- `L2TP`: L2TP 지원 여부. `1`이면 지원
- `UdpPort`: 서버가 별도로 노출한 UDP 포트 숫자
- `NumClients`: 현재 연결 클라이언트 수 추정치
- `NumLogin`: 누적 로그인 수
- `RecvSize / SendSize`: 누적 수신/송신 바이트
- `CreateDate / VerifyDate`: 밀리초 epoch 시간. 문서 표에서는 KST로 같이 표시
- `HostUniqueKey`: 서버 식별용 고유 키
- `Owner / Message / Abuse`: 운영자 설명, 메시지, abuse 안내 문자열
- `IsKernelModeNAT / Log / LogPermanent`: 커널 NAT 여부, 로그 지원 여부, 영구 로그 여부
- `Soft / Build / Os`: 서버 소프트웨어 문자열과 빌드/OS 코드
- `SpeedToJapan`: 일본 기준 속도 수치
- `Uptime`: 서버 업타임 밀리초

예를 들어 아래 한 줄은 이렇게 읽으면 된다.

- `public-vpn-51 / 219.100.37.13 / JP / Score 2955109 / SSL 443 / UDP 1195 1435`
- 해석: 일본(`JP`)에 있는 `219.100.37.13` 서버이고, SSL-VPN은 443 포트를 쓰며 OpenVPN UDP는 `1195 1435` 포트를 광고한다.
